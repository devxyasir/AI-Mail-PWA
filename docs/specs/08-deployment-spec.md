# 08 — Deployment Spec

## Goal

Deploy the app to Vercel with all features working in production.

## Pre-Deployment Checklist

- [ ] `npm run lint` — zero errors
- [ ] `npm run test` — zero failures
- [ ] `npm run build` — succeeds
- [ ] All environment variables set in Vercel
- [ ] OAuth redirect URIs updated for production URL
- [ ] PWA manifest is valid
- [ ] Supabase connection works from Vercel region

## Environment Variables Required

```
DATABASE_URL              # Supabase connection string
NEXTAUTH_SECRET           # Random 32+ char string
NEXTAUTH_URL              # https://your-app.vercel.app
GOOGLE_CLIENT_ID
GOOGLE_CLIENT_SECRET
MICROSOFT_CLIENT_ID
MICROSOFT_CLIENT_SECRET
MICROSOFT_TENANT_ID       # common
AI_API_KEY                # OpenRouter key
AI_BASE_URL               # https://openrouter.ai/api/v1
CREDENTIAL_ENCRYPTION_KEY # Random 32 char key
```

## Vercel Configuration

Build command: `npm run build`
Output directory: `.next`
Install command: `npm install`
Node version: 18.x or 20.x

## OAuth Redirect URIs

### Google Cloud Console

Authorized redirect URIs:
```
http://localhost:3000/api/auth/callback/google
https://your-app.vercel.app/api/auth/callback/google
```

### Azure App Registration

Redirect URIs:
```
http://localhost:3000/api/auth/callback/azure-ad
https://your-app.vercel.app/api/auth/callback/azure-ad
```

## PWA Requirements

- `public/manifest.json` with name, icons, display: standalone
- Icons at 192x192 and 512x512
- `next-pwa` configured in `next.config.js`
- Service worker generated by next-pwa

## Acceptance Criteria

- [ ] App is accessible at live Vercel URL
- [ ] Gmail login works in production
- [ ] Microsoft login works in production
- [ ] IMAP login works in production
- [ ] AI features work in production
- [ ] PWA install prompt appears on mobile
- [ ] No console errors in production
